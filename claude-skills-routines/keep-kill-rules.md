# Claude Code Skills: Keep / Kill Rules

Most Claude Code setups get bloated fast. Keep skills that save repeated work. Kill skills that sound clever but do not change output.

## Keep

- Skills that encode a real workflow you run weekly.
- Skills that point Claude to source-of-truth files.
- Skills that prevent repeated mistakes.
- Skills that include verification steps.
- Skills that produce a reviewable artifact.

## Kill

- Skills that are just style vibes.
- Skills you have not used in 30 days.
- Skills that duplicate another skill.
- Skills that make Claude ask more questions without doing better work.
- Skills that create outputs you never review or ship.

## Useful Routine Template

```text
Goal:
What should be true when this is done?

Inputs:
Which files, URLs, exports, or notes matter?

Rules:
What should never be changed or invented?

Workflow:
1. Read the source of truth.
2. Produce the smallest useful artifact.
3. Verify it.
4. Summarize changed files, proof, and blockers.
```
